## 서브넷 설계

> 192.168.200.0/24 → VLAN 3개 필요 → 2비트 빌림 → **/26**

| VLAN | 이름 | 네트워크 | 사용 가능 IP | 브로드캐스트 | 게이트웨이 |
|------|------|---------|------------|------------|----------|
| 100 | ccna | 192.168.200.0/26 | .1 ~ .62 | .63 | **192.168.200.1** |
| 200 | ccnp | 192.168.200.64/26 | .65 ~ .126 | .127 | **192.168.200.65** |
| 300 | ccie | 192.168.200.128/26 | .129 ~ .190 | .191 | **192.168.200.129** |

**서브넷 계산 원리**
- VLAN 3개 필요 → 2² = 4 ≥ 3 → **2비트 빌림**
- /24 + 2 = **/26** → 서브넷마스크 **255.255.255.192**
- 11000000 = 128 + 64 = **192**
- 남은 6비트 → 2⁶ = **64씩 분할**

## 설정 명령어

### SW-Left

```bash
enable
conf t

# VLAN 생성
vlan 100
name ccna
exit
vlan 200
name ccnp
exit
vlan 300
name ccie
exit

# 포트 배정
interface fa0/1
switchport mode access
switchport access vlan 100
exit

interface fa0/2
switchport mode access
switchport access vlan 300
exit

# 트렁크 설정
interface fa0/3
switchport mode trunk
exit

interface fa0/4
switchport mode trunk
exit
end
```

### Router (Inter-VLAN Routing)

```bash
enable
conf t

# 물리 인터페이스 활성화
interface gig0/0
no shutdown
exit

# 서브인터페이스 설정 (게이트웨이)
interface gig0/0.100
encapsulation dot1q 100
ip address 192.168.200.1 255.255.255.192
exit

interface gig0/0.200
encapsulation dot1q 200
ip address 192.168.200.65 255.255.255.192
exit

interface gig0/0.300
encapsulation dot1q 300
ip address 192.168.200.129 255.255.255.192
exit
end
```

## 핵심 개념 정리

### VLAN이란?
물리적인 네트워크와 별도로 **가상의 네트워크**를 만들어 보안성과 관리성을 높이는 기술

### 트렁크(Trunk)란?
여러 VLAN의 데이터를 하나의 케이블로 전송하는 **전용 통로** (802.1Q 태그 사용)

### 서브인터페이스란?
라우터의 물리 포트 1개를 **가상으로 여러 개로 쪼개는** 기술  
→ `encapsulation dot1q [VLAN번호]` 로 각 VLAN 전용 통로 지정

이게 바로 오늘 배운 Router-on-a-Stick!
"""
