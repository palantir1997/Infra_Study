## 1. 기본 개념 정리

| 용어 | 설명 | 비유 |
|------|------|------|
| **DTE** (Data Terminal Equipment) | 데이터를 생성하거나 최종 소비하는 장치 (PC, 서버, 라우터) | 전화를 거는 사람 |
| **DCE** (Data Circuit-terminating Equipment) | DTE와 통신 회선 사이에서 변환/중계하는 장치 (모뎀, CSU/DSU) | 전화 교환기 |

> **시리얼 케이블 연결 시 주의**
> DCE 쪽(시계 모양 아이콘) 라우터에서 반드시 `clock rate` 설정 필요!
> ```
> show controllers [인터페이스]  ← DCE/DTE 확인 명령어
> ```

## 2. 초기 IP 세팅

> **핵심 원칙**: 라우터 장치 추가 시 → 전원 OFF → 추가 → 전원 ON

### 네트워크 구성도 요약

```
[PC]──────[R-A]────────────────────[R-B]────────────────[R-C]──────[Server]
          Se0/3/0: 10.10.10.10/24    .20                 .40        192.168.200.254
          Se0/3/1: 30.30.30.30/24    .60                             Gig0/0
          Fa0/1: 192.168.100.254                 Se0/3/1: 20.20.20.40
```

### 각 라우터 인터페이스 주소

**Router A (왼쪽)**
```
Fa0/1     → 192.168.100.254/24   (PC 쪽)
Se0/3/0   → 10.10.10.10/24      (R-B 기본 경로)
Se0/3/1   → 30.30.30.30/24      (R-B 우회 경로)
```

**Router B (가운데)**
```
Se0/3/0   → 10.10.10.20/24
Se0/2/0   → 30.30.30.60/24
Se0/3/1   → 20.20.20.20/24      (R-C 쪽)
Gig0/0    → 192.168.150.254     (추가 PC 쪽)
```

**Router C (오른쪽)**
```
Se0/3/0   → 20.20.20.40/24      (R-B 쪽)
Gig0/0    → 192.168.200.254/24  (서버 쪽)
```

## 3. 정적 라우팅 (Static Route)

> **개념**: 관리자가 직접 경로를 지정해주는 방식
> `show ip route` 에서 **S** 로 표시됨

### 명령어 공식

```
ip route [목적지 네트워크] [서브넷 마스크] [다음 거쳐갈 이웃 IP]
```

### 각 라우터별 설정

**Router A**
```cisco
R-A(config)# ip route 192.168.150.0 255.255.255.0 10.10.10.20
R-A(config)# ip route 192.168.200.0 255.255.255.0 10.10.10.20
```

**Router B**
```cisco
R-B(config)# ip route 192.168.200.0 255.255.255.0 20.20.20.40
```

**Router C**
```cisco
R-C(config)# ip route 192.168.100.0 255.255.255.0 20.20.20.20
R-C(config)# ip route 192.168.150.0 255.255.255.0 20.20.20.20
```

### 설정 확인 및 저장
```cisco
R-A# show ip route          ← 라우팅 테이블 확인
R-A# write memory           ← 설정 저장 (중요!)
R-A# do sh run              ← 전체 설정 보기
```

### `show ip route` 결과 해석

```
S    192.168.150.0/24 [1/0] via 10.10.10.20
│                      │ └─ Metric (코스트, 낮을수록 좋음)
│                      └─── AD 값 (Administrative Distance, 낮을수록 신뢰)
└── S = Static (정적 경로)    C = Connected (직접 연결)
```

## 4. 로드 밸런싱

> **개념**: 같은 목적지에 경로가 2개 이상일 때, 트래픽을 여러 경로로 분산하는 것

### 추가 라우터 연결 후 우회로 설정

```cisco
R-A(config)# ip route 192.168.150.0 255.255.255.0 30.30.30.60  ← 우회로 추가
R-B(config)# ip route 192.168.100.0 255.255.255.0 30.30.30.30
```

### 결과 - 라우팅 테이블에 같은 목적지로 2줄이 올라옴

```
S    192.168.150.0/24 [1/0] via 10.10.10.20   ← 기본 경로
                      [1/0] via 30.30.30.60   ← 우회 경로 (로드 밸런싱!)
S    192.168.200.0/24 [1/0] via 10.10.10.20
```

> 이처럼 **동일 목적지에 경로 2개** = **로드 밸런서 역할**

## 5. AD값과 우선순위

> **AD (Administrative Distance)**: 경로의 신뢰도. **낮을수록 우선순위 높음**
> `[1/0]`에서 앞 숫자 = **AD값**, 뒷 숫자 = **Metric(코스트)**

### AD 기본값 비교표

| 프로토콜 | AD 값 | 신뢰도 |
|----------|--------|--------|
| Connected (직접 연결) | **0** | 1순위 (최고) |
| Static Route (정적) | **1** | 2순위 |
| EIGRP | **90** | 3순위 |
| OSPF | **110** | 4순위 |
| RIP | **120** | 5순위 |

### AD값 수동 변경 (특정 경로에 낮은 우선순위 부여)

```cisco
R-A(config)# ip route 192.168.150.0 255.255.255.0 30.30.30.60 10
```
> 맨 끝 `10` = 이 경로의 AD값을 10으로 변경 → 다른 경로보다 선호도 조정 가능

### AD값 변경 결과 - 우회로 제거됨

```
S    192.168.150.0/24 [1/0] via 10.10.10.20   ← AD 1 (우선)
S    192.168.200.0/24 [1/0] via 10.10.10.20
```
> 30.30.30.60 경로는 AD=10이라 기존 경로(AD=1)보다 낮은 우선순위가 되어 사라짐

## 6. 경로 추적 및 장애 테스트

### tracert (경로 추적 명령어)

```
C:\> tracert 192.168.150.100
```

**정상 시 결과** (기본 경로 사용)
```
1   2ms    192.168.100.254   ← R-A (게이트웨이)
2   0ms    10.10.10.20       ← R-B (중간 라우터)
3   0ms    192.168.150.100   ← 목적지 도달
```

**장애 발생 후 결과** (`shutdown`으로 Se0/3/0 다운 시)
```
1   1ms    192.168.100.254   ← R-A (게이트웨이)
2   12ms   30.30.30.60       ← R-B (우회 경로로 자동 전환! )
3   0ms    192.168.150.100   ← 목적지 도달
```

> **포인트**: 장애 발생 시 고속선 대신 우회선(30.30.30.60)을 자동으로 사용!

## 7. 동적 라우팅 - RIP

> **개념**: 라우터끼리 자동으로 경로를 교환하는 프로토콜
> 기존 static 경로는 `no ip route ...`로 모두 제거 후 시작

### RIP 설정 방법

```cisco
R-A(config)# router rip
R-A(config-router)# network 192.168.100.0
R-A(config-router)# network 10.10.10.0
R-A(config-router)# network 30.30.30.0
R-A(config-router)# no auto-summary        ← 자동 축약 방지 (중요!)
```

### RIP vs OSPF 비교

| 항목 | RIP | OSPF |
|------|-----|------|
| 설정 난이도 | 쉬움 | 복잡함 |
| 경로 선택 기준 | 거쳐가는 라우터 수 (Hop Count) | 대역폭 기반 최적 경로 |
| 최대 홉 수 | **15홉** (16부터 도달 불가) | 제한 없음 |
| 변경 감지 속도 | 느림 | 빠름 (즉시 반응) |
| 리소스 사용 | 적음 | CPU/메모리 많이 사용 |
| 대규모 네트워크 | 부적합 | 적합 (Area 개념으로 분리) |
| 비유 | 신호등 없는 시골길이라도 홉 수 적으면 선택 | 8차선 고속도로를 기가 막히게 찾아냄 |

## 8. 동적 라우팅 - EIGRP

> **개념**: Cisco 전용 프로토콜. RIP의 한계 + OSPF의 복잡함을 절충한 프로토콜
> AD값 90, 대역폭 + 지연시간 기반으로 최적 경로 계산

### EIGRP 설정 방법

```cisco
R-A(config)# router eigrp 10                          ← 프로세스 번호 (같아야 통신됨)
R-A(config-router)# network 192.168.100.0 0.0.0.255   ← 와일드카드 마스크 사용!
R-A(config-router)# network 10.10.10.0 0.0.0.255
R-A(config-router)# network 30.30.30.0 0.0.0.255
```

> **와일드카드 마스크란?**
> 서브넷 마스크를 비트 반전한 값!
> 255.255.255.0 → 비트 반전 → **0.0.0.255**
> (1→0, 0→1 로 바꾸면 됨)

### EIGRP Metric 계산 공식

$$Metric = [K_1 \times BW + K_3 \times Delay] \times 256$$

```
K1 (Bandwidth) = 1   → 대역폭 중요! 
K2 (Load)      = 0   → 부하 무시
K3 (Delay)     = 1   → 지연시간 중요! 
K4, K5         = 0   → 신뢰성 무시
```
> `sh ip protocol` 명령어로 K값 확인 가능

### 이웃 관계 성립 확인 - 핵심 로그

```
%DUAL-5-NBRCHANGE: IP-EIGRP 10: Neighbor 10.10.10.10 (Serial0/3/0) is up: new adjacency
```
> 이 메시지가 뜨면 EIGRP 이웃(Neighbor) 연결 성공! 

### 라우팅 테이블 결과 (EIGRP)

```
D    192.168.150.0/24 [90/2172416] via 10.10.10.20, 00:03:28, Serial0/3/0
                      [90/2172416] via 30.30.30.60, 00:03:28, Serial0/3/1
```
> `D` = EIGRP 경로, `[90/2172416]` = [AD값/Metric]

### EIGRP 상세 확인 명령어

```cisco
R-B# sh ip eigrp neighbors    ← 이웃 라우터 목록
R-B# sh ip eigrp topology     ← 전체 토폴로지 테이블
R-B# sh ip eigrp interfaces   ← EIGRP 활성 인터페이스
R-B# sh ip eigrp traffic      ← 트래픽 통계
```

## 9. 자주 쓰는 명령어 모음

| 명령어 | 용도 |
|--------|------|
| `show ip route` | 라우팅 테이블 확인 (C/S/D/R 구분) |
| `show ip protocol` | 현재 동작 중인 라우팅 프로토콜 확인 |
| `show running-config` / `do sh run` | 현재 설정 전체 보기 |
| `show controllers [인터페이스]` | DCE/DTE 구분 확인 |
| `show interface se0/3/0` | 인터페이스 상세 정보 |
| `write memory` | 설정 저장 |
| `tracert [IP]` (Windows) | 경로 추적 |
| `no auto-summary` | 자동 축약 비활성화 |
| `shutdown` / `no shutdown` | 인터페이스 활성/비활성 |

## 오늘의 핵심 정리

```
라우팅 프로토콜 분류
├── Static (정적)   : 관리자가 직접 경로 설정
└── Dynamic (동적)  : 라우터끼리 자동으로 경로 교환
    ├── RIP    : 홉 수 기반, 소규모 네트워크
    ├── EIGRP  : Cisco 전용, 대역폭+지연 기반
    ├── OSPF   : 표준 프로토콜, 대규모 네트워크
    └── BGP    : 인터넷 전체 라우팅 (AS 간)
```

> **가장 중요한 것**: `show ip route` 결과 해석!
> - `C` (Connected): 직접 연결된 인터페이스
> - `S` (Static): 수동으로 지정한 경로
> - `D` (EIGRP): EIGRP가 찾아준 경로
> - `R` (RIP): RIP가 찾아준 경로
