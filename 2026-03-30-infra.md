## 1. ACL (Access Control List) 개요

ACL은 라우터에서 패킷을 허용하거나 차단하는 규칙 목록이다.  
번호 범위에 따라 Standard와 Extended로 구분된다.

| 번호 범위 | 종류 | 특징 |
|-----------|------|------|
| 1 ~ 99 | Standard ACL | 출발지 IP 주소만 확인 (단순) |
| 100 ~ 199 | Extended ACL | 출발지, 목적지, 프로토콜, 포트 번호까지 확인 (정교) |

---

## 2. Standard ACL

출발지 IP만 기준으로 허용/차단한다.  
적용 위치: **목적지 인터페이스에 최대한 가깝게 (out 방향)** 적용하는 것이 원칙.

### 기본 설정 예시

```
Router(config)# access-list 10 permit host 192.168.0.10
Router(config)# access-list 10 deny 192.168.0.0 0.0.0.255
Router(config)# access-list 10 permit any

Router(config-if)# ip access-group 10 out
```

### 확인 명령어

```
Router# show access-list

Standard IP access list 10
  10 permit host 192.168.0.10
  20 deny 192.168.0.0 0.0.0.255
  30 permit any
```

### ACL 삭제

```
Router(config)# no access-list 10
```

---

## 3. Extended ACL

출발지/목적지 IP, 프로토콜(TCP/UDP/ICMP), 포트 번호까지 모두 지정 가능하다.  
적용 위치: **출발지 인터페이스에 최대한 가깝게 (in 방향)** 적용하는 것이 원칙.

### 명령어 구조

```
access-list [번호] [permit|deny] [프로토콜] [출발지] [목적지] [포트조건]
```

### 포트 조건 키워드

| 키워드 | 의미 |
|--------|------|
| `eq` | 특정 포트 번호와 일치 |
| `gt` | 특정 포트 번호보다 큰 경우 |
| `lt` | 특정 포트 번호보다 작은 경우 |
| `neq` | 특정 포트 번호가 아닌 경우 |
| `range` | 포트 번호 범위 지정 |

### 설정 예시

```
# 192.168.0.10 -> 192.168.1.100 으로 가는 ping echo-reply 차단
Router(config)# access-list 100 deny icmp host 192.168.0.10 host 192.168.1.100 echo-reply

# 192.168.0.10 -> 192.168.1.100 으로 가는 HTTP(80) 허용
Router(config)# access-list 100 permit tcp host 192.168.0.10 host 192.168.1.100 eq www

# 192.168.0.0/24 -> 192.168.1.100 으로 가는 HTTP(80) 차단
Router(config)# access-list 100 deny tcp 192.168.0.0 0.0.0.255 host 192.168.1.100 eq 80
```

## 4. Named ACL

번호 대신 이름으로 ACL을 관리한다.  
가독성이 높고 특정 라인만 삭제/수정이 가능하다.

### Standard Named ACL

```
Router(config)# ip access-list standard eacl
Router(config-std-nacl)# permit host 192.168.0.10
Router(config-std-nacl)# deny 192.168.0.0 0.0.0.255
Router(config-std-nacl)# permit any

Router(config-if)# ip access-group eacl out
```

### Extended Named ACL

```
Router(config)# ip access-list extended acltest
Router(config-ext-nacl)# deny tcp host 192.168.1.100 host 192.168.0.200 eq ftp
Router(config-ext-nacl)# permit icmp host 192.168.1.100 host 192.168.0.200
Router(config-ext-nacl)# permit tcp 192.168.1.0 0.0.0.255 host 192.168.0.200 eq ftp
Router(config-ext-nacl)# deny icmp host 192.168.1.200 host 192.168.0.10
Router(config-ext-nacl)# permit ip any any
```

## 5. ACL 실습 예제

### 요구사항

| 조건 | 내용 |
|------|------|
| 조건 1 | 192.168.1.100 -> 192.168.0.200 FTP 차단 (ICMP는 허용) |
| 조건 2 | 192.168.1.0/24 -> 192.168.0.200 FTP 허용 (1.100 제외) |
| 조건 3 | 192.168.1.200 -> 192.168.0.10 ICMP(Ping) 차단 |
| 조건 4 | 그 외 모든 트래픽 허용 |

### 설정

```
Router(config)# access-list 101 deny tcp host 192.168.1.100 host 192.168.0.200 eq ftp
Router(config)# access-list 101 permit tcp 192.168.1.0 0.0.0.255 host 192.168.0.200 eq ftp
Router(config)# access-list 101 deny icmp host 192.168.1.200 host 192.168.0.10
Router(config)# access-list 101 permit ip any any
```

> ACL은 위에서 아래로 순차 적용된다.  
> 마지막에 `permit ip any any`가 없으면 묵시적 deny all에 의해 모든 트래픽이 차단된다.

## 6. NAT / PAT

### 개념 비교

| 기술 | 핵심 개념 | 비유 |
|------|-----------|------|
| Static NAT | 사설 IP 1개 : 공인 IP 1개 (1:1 고정 매핑) | 개인 우편함 하나에 고정 주소 |
| Dynamic NAT | 사설 IP 여러 개 : 공인 IP 풀 (N:N 동적 매핑) | 공유 우편함 선착순 배정 |
| PAT (Overload) | 공인 IP 1개 : 사설 IP 여러 개 (포트로 구별) | 아파트 1동 주소에 호수로 구별 |

### Dynamic NAT 설정

```
# 1. 변환 대상 내부 주소 지정 (ACL)
Router(config)# access-list 10 permit 192.168.1.0 0.0.0.255

# 2. 공인 IP 풀 생성
Router(config)# ip nat pool nattest 20.20.20.1 20.20.20.254 netmask 255.255.255.0

# 3. NAT 매핑 선언
Router(config)# ip nat inside source list 10 pool nattest

# 4. 인터페이스 방향 지정
Router(config)# interface gi0/0
Router(config-if)# ip nat inside

Router(config)# interface se0/3/1
Router(config-if)# ip nat outside
```

### Static NAT 설정

```
Router(config)# ip nat inside source static 192.168.1.100 20.20.20.1
```

### PAT 설정 (overload)

```
# ip nat inside source 명령 끝에 overload 추가
Router(config)# ip nat inside source list 10 pool nattest overload
```

`overload` 키워드를 붙이면 공인 IP 1개를 포트 번호로 다중 매핑하는 PAT로 동작한다.

### 변환 테이블 확인

```
Router# show ip nat translations
```

## 7. IPv6

### 주소 체계

IPv6 주소는 128비트, 콜론(:)으로 구분된 8개의 16진수 블록으로 표현한다.

```
2001:0001:0001:0001:0000:0000:0000:0010
```

연속된 0 블록은 `::` 로 축약 가능하며, 각 블록 앞의 0도 생략 가능하다.

### 주소 유형

| 유형 | 범위 | 설명 |
|------|------|------|
| Global Unicast | `2001::/16`, `2002::/16`, `2003::/18`, `3FFE::/18` | 인터넷에서 사용하는 공인 주소 |
| Link-Local | `FE80::/10` | 동일 링크(세그먼트) 내에서만 사용 |
| Site-Local | `FEC0::/10` | 특정 사이트 내에서만 사용 |

### 인터페이스 IPv6 주소 설정

```
Router(config-if)# ipv6 address 2001:1:1:1::1/64
```

`2001:1:1:1` 네트워크의 호스트 `1`번으로 설정, 네트워크 범위는 64비트.

### IPv6 라우팅 활성화 및 OSPF 설정

```
# IPv6 유니캐스트 라우팅 전역 활성화
Router(config)# ipv6 unicast-routing

# 인터페이스에 OSPFv3 영역 지정 (인터페이스마다 반복)
Router(config-if)# ipv6 ospf 1 area 0
```

### IPv6 라우팅 테이블 확인

```
Router# show ipv6 route
```

#### 결과 예시

```
O  2001:1:1:1::/64 [110/129]
     via FE80::260:3EFF:FE50:5C56, Serial0/3/1
C  2001:4:4:4::/64 [0/0]
     via GigabitEthernet0/0, directly connected
L  2001:4:4:4::1/128 [0/0]
     via GigabitEthernet0/0, receive
```

#### 코드 의미

| 코드 | 의미 |
|------|------|
| `C` | Connected (직접 연결) |
| `L` | Local (자기 자신의 주소) |
| `O` | OSPF intra-area |
| `OI` | OSPF inter-area |
| `S` | Static |

## 핵심 정리

```
ACL 구조
├── Standard (1~99)    : 출발지 IP만 확인 → 목적지 인터페이스 근처에 적용
└── Extended (100~199) : 출발지, 목적지, 프로토콜, 포트 번호까지 확인 → 출발지 인터페이스 근처에 적용

NAT 종류
├── Static NAT  : 1:1 고정 매핑
├── Dynamic NAT : IP 풀에서 동적 할당
└── PAT         : 포트 번호로 다중 매핑 (overload)

IPv6 주소 유형
├── Global Unicast : 2001::/16 등 (공인)
├── Link-Local     : FE80::/10 (링크 내부)
└── Site-Local     : FEC0::/10 (사이트 내부)
```
