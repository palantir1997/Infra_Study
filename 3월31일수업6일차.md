## 1. GRE 터널 기본 설정

GRE(Generic Routing Encapsulation)는 패킷을 캡슐에 넣어서 전달하는 터널링 기술이다.

### 본사 라우터

```
int tunnel 100
ip addr 10.10.10.10 255.255.255.0
tunnel mode gre ip
tunnel source se0/3/0
tunnel destination 2.2.2.2
ip route 172.16.0.0 255.255.255.0 10.10.10.20
```

### 지사 라우터

```
int tunnel 100
ip addr 10.10.10.20 255.255.255.0
tunnel mode gre ip
tunnel source se0/3/1
tunnel destination 1.1.1.1
ip route 192.168.100.0 255.255.255.0 10.10.10.10
```

### tracert 결과 확인

```
C:\>tracert 192.168.100.10

Tracing route to 192.168.100.10 over a maximum of 30 hops:

1    0 ms    0 ms    0 ms  172.16.0.1
2    2 ms    2 ms    9 ms  10.10.10.10
3    2 ms    2 ms    2 ms  192.168.100.10
```

## 2. IPSec VPN 설정

### 본사 라우터 설정

```
access-list 100 permit gre host 1.1.1.1 host 2.2.2.2
access-list 100 permit gre 192.168.100.0 0.0.0.255 host 2.2.2.2

crypto isakmp policy 10
 authentication pre-share
 encryption 3des
 hash md5

crypto isakmp key 1234 address 2.2.2.2
crypto ipsec transform-set test esp-3des esp-md5-hmac

crypto map vpn-map 10 ipsec-isakmp
 match address 100
 set peer 2.2.2.2
 set transform-set test

interface se0/3/0
 crypto map vpn-map
```

### 지사 라우터 설정

```
access-list 100 permit gre host 2.2.2.1 host 1.1.1.1
access-list 100 permit gre 172.16.0.0 0.0.0.255 host 1.1.1.1

crypto isakmp policy 10
 authentication pre-share
 encryption 3des
 hash md5

crypto isakmp key 1234 address 1.1.1.1
crypto ipsec transform-set test esp-3des esp-md5-hmac

crypto map vpn-map 10 ipsec-isakmp
 match address 100
 set peer 1.1.1.1
 set transform-set test

interface se0/3/1
 crypto map vpn-map
```

> key 값은 양쪽 라우터가 동일해야 한다.

### Transform-set 확인

```
Router# show crypto ipsec transform-set

Transform set test: { esp-3des esp-sha-hmac }
 will negotiate = { Tunnel, },
```

### Telnet 원격 접속 설정

```
Router(config)# enable password 1234
Router(config)# line vty 0 4
Router(config-line)# login
Router(config-line)# password 1234
```

```
C:\>telnet 192.168.100.1

Password:

```

## 4. 라우터별 핵심 설정 명령어

### 본사 라우터 (HQ Router)

```
interface gi0/0
 ip address 192.168.200.1 255.255.255.0
 no shutdown

interface se0/3/0
 ip address 5.5.5.1 255.255.255.0
 no shutdown

interface tunnel 0
 ip address 10.10.10.1 255.255.255.0
 tunnel source se0/3/0
 tunnel destination 6.6.6.2

router ospf 1
 network 192.168.200.0 0.0.0.255 area 0
 network 10.10.10.0 0.0.0.255 area 0
 network 5.5.5.0 0.0.0.255 area 0

! ACL 설정
! PC F(172.16.10.2)가 Server B(192.168.200.3)에 HTTP(80) 접속 허용
access-list 100 permit tcp host 172.16.10.2 host 192.168.200.3 eq 80

! PC F에서 본사 내부로 ICMP 차단
access-list 100 deny icmp host 172.16.10.2 host 192.168.200.2
access-list 100 deny icmp host 172.16.10.2 host 192.168.200.3

! 지사 PC D에서 Server B로 FTP(21) 차단
access-list 100 deny tcp host 192.168.250.2 host 192.168.200.3 eq 21

! 나머지 트래픽 전체 허용
access-list 100 permit ip any any

interface se0/3/0
 ip access-group 100 in
```

### 중앙 라우터 (Core Router - NAT 담당)

```
interface gi0/0
 ip address 172.16.10.1 255.255.255.0
 ip nat inside
 no shutdown

interface se0/3/0
 ip address 5.5.5.2 255.255.255.0
 ip nat outside
 no shutdown

interface se0/3/1
 ip address 6.6.6.1 255.255.255.0
 ip nat outside
 no shutdown

! NAT 설정: 172.16.10.0 대역을 외부 인터페이스로 변환 (PAT)
access-list 1 permit 172.16.10.0 0.0.0.255
ip nat inside source list 1 interface se0/3/0 overload

router ospf 1
 network 5.5.5.0 0.0.0.255 area 0
 network 6.6.6.0 0.0.0.255 area 0
 network 172.16.10.0 0.0.0.255 area 0
```

### 지사 라우터 (Branch Router)

```
interface gi0/0
 ip address 192.168.250.1 255.255.255.0
 no shutdown

interface se0/3/1
 ip address 6.6.6.2 255.255.255.0
 no shutdown

interface tunnel 0
 ip address 10.10.10.2 255.255.255.0
 tunnel source se0/3/1
 tunnel destination 5.5.5.1

router ospf 1
 network 192.168.250.0 0.0.0.255 area 0
 network 10.10.10.0 0.0.0.255 area 0
 network 6.6.6.0 0.0.0.255 area 0
```

## 5. 최종 확인 명령어

| 명령어 | 확인 내용 |
|--------|---------|
| `show ip interface brief` | 인터페이스 상태 확인 (Tunnel0 포함 up/up 여부) |
| `show ip route` | 라우팅 테이블 확인 (O 표시 = OSPF 정상 작동) |
| `show ip ospf neighbor` | OSPF 이웃 라우터 목록 확인 |
| `show ip nat translations` | NAT 변환 테이블 확인 |
| `show access-lists 100` | ACL 차단 횟수 확인 (matches 숫자 증가 여부) |
